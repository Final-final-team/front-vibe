# 0014. 마일스톤 목록 검색/필터 및 동적 위험 기준 도입

- 날짜: 2026-03-17

## 배경
마일스톤 개수가 늘어나면 현재 필요한 항목(위험/검토 대기/완료)만 빠르게 추려보기 어렵고,
기존 위험 판정이 고정 날짜 기준이라 시간이 지나면 의미가 약해진다.

## 선택
- 마일스톤 목록에 검색(이름/요약) + 상태 필터(전체/위험/검토 대기/완료) 추가
- 카드에 `다음 액션` 요약 노출
- 위험 판정 기준을 고정 날짜에서 "기한 3일 이내"의 동적 기준으로 변경

## 장점
- 운영자가 즉시 대응할 목록을 빠르게 좁힐 수 있음
- UI에서 다음 액션 컨텍스트를 바로 파악 가능
- 시간 경과에 따라 위험 기준이 자연스럽게 유지됨

## 단점
- 필터/검색 상태 관리가 추가되어 컴포넌트 복잡도가 소폭 증가
- 3일 기준이 프로젝트 상황에 따라 과하거나 약할 수 있음

## 후속
- 필요 시 위험 기준(예: 1일/3일/7일)을 설정값으로 승격
- 리스트가 더 커지면 서버 필터링 API 검토
