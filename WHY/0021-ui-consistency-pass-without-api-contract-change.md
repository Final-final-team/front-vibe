# UI 일관성 정리 패스

## 질문 배경
- 화면마다 상단 섹션 톤, 버튼 문법, 모달 크기, 뷰 전환 방식이 달라 보였다.
- 특히 업무 화면의 뷰 전환과 범위 전환이 부자연스럽고, 검토/미연동 화면의 카드 밀도도 들쭉날쭉했다.
- 조건은 명확했다.
  - API 요청 경로, 응답 타입, 주고받는 값은 바꾸지 않는다.
  - UI/UX 레이어만 정리한다.

## 선택지

### 1. 페이지별로 개별 수정
- 장점
  - 빠르게 특정 화면만 손볼 수 있다.
  - 코드 변경 범위가 작다.
- 단점
  - 다음 화면으로 넘어가면 다시 톤이 어긋난다.
  - 버튼, 모달, 인트로 섹션의 기준이 생기지 않는다.

### 2. 공통 시각 규칙을 먼저 만들고 주요 화면에 적용
- 장점
  - 상단 인트로, 모달 프레임, 세그먼트 버튼 같은 반복 UI가 통일된다.
  - 이후 페이지를 추가해도 같은 규칙으로 확장하기 쉽다.
- 단점
  - 처음에는 공통 컴포넌트와 페이지 수정이 같이 들어가서 작업량이 늘어난다.

## 이 프로젝트 기준 추천
- `2번`이 맞다.
- 현재 프로젝트는 기능 면적이 넓고, 업무/검토/멤버/권한/미연동 안내가 하나의 제품 안에서 이어진다.
- 그래서 화면 하나만 예쁘게 만드는 것보다, 반복 패턴을 통일하는 쪽이 UX 품질에 더 직접적이다.

## 최종 선택
- 공통 페이지 인트로 컴포넌트 추가
- 모달 패딩, 최대폭, 사이드 패널 경계 규칙 통일
- 업무 화면의 뷰 스위치와 범위 스위치를 pill/segment 문법으로 통일
- 미연동 화면도 같은 상단 톤으로 정리
- API 계약은 변경하지 않음

## 판단 근거
- 사용자는 시각적 부자연스러움과 통일감 부족을 직접 지적했다.
- 화면별 하드코딩 회색/파란색 스타일이 섞여 있어서 토큰 기반 색상으로 맞출 필요가 있었다.
- API를 건드리지 않는 조건이 있기 때문에, 공통 UI 프레임만 정리하는 것이 가장 안전한 개선 경로였다.
