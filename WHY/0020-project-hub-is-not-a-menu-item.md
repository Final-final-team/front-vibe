# 0020. 프로젝트 허브는 메뉴 항목이 아니라 상위 페이지

## 질문 배경
- 집 아이콘을 눌렀을 때 보이는 프로젝트 카드 화면은 메인 허브인데, 현재는 메뉴 안의 `홈`처럼 보여서 위계가 어색하다.
- 사용자는 먼저 프로젝트를 고르고, 그 다음에야 업무/검토 같은 하위 메뉴를 보길 기대한다.

## 선택지
### 1. `홈`을 메뉴 항목으로 유지한다
- 장점
  - 구현이 단순하다.
  - 다른 메뉴와 같은 방식으로 이동할 수 있다.
- 단점
  - 프로젝트 허브가 업무/검토와 같은 레벨처럼 보인다.
  - 정보 구조가 평평해져서 상위 개념이 약해진다.

### 2. 프로젝트 허브를 상위 페이지로 두고 메뉴에서 뺀다
- 장점
  - 집 아이콘이 상위 이동의 의미를 분명하게 가진다.
  - 프로젝트를 고른 뒤에만 업무/검토 메뉴가 나타나는 구조가 자연스럽다.
  - 사용자가 프로젝트 선택과 기능 탐색을 단계적으로 이해할 수 있다.
- 단점
  - 허브 화면과 프로젝트 내부 화면의 레이아웃 차이를 조금 더 관리해야 한다.

## 이 프로젝트 기준 권고
- 현재는 `2. 프로젝트 허브를 상위 페이지로 두고 메뉴에서 빼는 방식`이 맞다.

## 최종 선택
- 집 아이콘은 `/projects` 허브로만 연결한다.
- 사이드바 메뉴에서는 `홈` 항목을 제거한다.
- `/projects` 허브에서는 프로젝트 카드만 중심으로 보여주고, 프로젝트를 선택한 뒤에야 업무/검토 메뉴를 사용하게 한다.

## 선택 근거
- 프로젝트 허브는 기능 메뉴가 아니라 프로젝트 문맥을 선택하는 상위 단계다.
- 업무/검토는 특정 프로젝트를 고른 뒤에 열리는 하위 기능이므로, 동일 메뉴 레벨로 두는 것이 오히려 구조를 흐린다.
