# 0012. 업무/검토 화면을 플랫한 협업툴 레이아웃으로 전환

- 상태: 승인됨
- 날짜: 2026-03-17

## 맥락

기존 업무/검토 화면은 카드형 대시보드 요소가 많아 실제 작업 데이터보다 장식성 박스가 먼저 보였다.  
특히 내부 업무관리툴 관점에서는 다음 문제가 반복됐다.

- 상단 히어로와 메트릭 카드가 실제 테이블/큐보다 더 큰 면적을 차지했다.
- 상태, 설명, 액션이 모두 박스 단위로 분절되어 한눈에 흐름이 잡히지 않았다.
- 협업툴이라기보다 API 명세나 데모 대시보드처럼 보였다.

이 프로젝트의 핵심은 공개 페이지가 아니라 로그인 후 내부 사용자가 `업무`, `검토`, `멤버`, `권한`을 빠르게 다루는 작업 화면이다.

## 검토한 선택지

### 1. 기존 카드형 레이아웃 유지 + 색상만 조정

- 구현 비용은 가장 낮다.
- 하지만 사용자가 느끼는 “박스가 많고 실제 내용이 늦게 나온다”는 문제를 해결하지 못한다.

### 2. 메트릭 카드만 줄이고 전체 구조는 유지

- 일부 높이는 줄일 수 있다.
- 그러나 정보 전달 방식 자체가 카드 중심이라 업무툴다운 밀도와 흐름은 부족하다.

### 3. 플랫한 협업툴 레이아웃으로 전환

- 상단은 텍스트/라인 중심 크롬으로 축소한다.
- 주요 정보는 리스트, 테이블, 분할 패널, 언더라인 탭처럼 선형 구조로 보여준다.
- 요약 정보도 별도 카드 대신 인라인 통계로 표현한다.

## 결정

`업무`와 `검토` 메인 화면은 플랫한 협업툴 레이아웃으로 전환한다.

구체 원칙은 다음과 같다.

- 큰 히어로 카드, 설명 카드, 장식용 메트릭 카드를 줄인다.
- 실제 업무 데이터와 검토 큐가 첫 화면 상단에서 바로 보이게 한다.
- 상단 크롬은 `제목 + 프로젝트 선택 + 핵심 수치 + 뷰 전환` 정도만 남긴다.
- 테이블/칸반/캘린더/차트/간트는 같은 공통 뷰 모델을 공유하고, 표시 방식만 달리한다.
- 선택된 업무의 상세는 별도 카드 더미가 아니라 우측 분할 패널로 보여준다.
- 라운드 배지, 상태 pill, 뷰 탭도 `rounded-full` 위주 표현 대신 더 낮은 반경과 얇은 보더를 사용한다.

## 결과

장점:

- 실제 업무/검토 데이터가 더 빨리 보인다.
- 협업툴다운 밀도와 흐름이 살아난다.
- 이후 `멤버`, `역할/권한`, `마일스톤`도 같은 표현 규칙으로 맞추기 쉬워진다.

단점:

- 기존 카드형 레이아웃보다 시각적으로 덜 친절하게 느껴질 수 있다.
- 차트/캘린더/간트는 향후 실제 백엔드 필드 매핑이 들어올 때 세부 조정이 필요하다.
