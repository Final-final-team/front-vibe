# 백엔드 연동 대기 항목

현재 프론트는 [be-api-server](https://github.com/Final-final-team/be-api-server) `dev` 브랜치를 기준으로 실제 연결 가능한 화면만 메인 동선에 노출합니다.

## 현재 메인 UI에서 제거한 항목

### 감사 로그
- 프로젝트 전역 감사 로그 전용 조회 API가 아직 없습니다.
- 현재 백엔드는 review 이력 중심 데이터는 제공하지만, 프로젝트 단위 전역 로그 화면을 구성할 전용 엔드포인트가 없습니다.
- 따라서 사이드바와 라우트에서 제거했습니다.
- 향후 API가 추가되면 `감사 로그`를 별도 운영 화면으로 복구합니다.

## 구현은 되어 있으나 추후 실제 API 매핑이 필요한 항목

### 역할 권한 편집
- 현재는 mock 권한 카탈로그 기준으로 역할별 허용/미허용을 편집합니다.
- 향후 역할별 권한 추가/제거 API가 들어오면 `권한 관리` 모달을 저장 지점으로 그대로 연결합니다.

### 마일스톤 생성
- 새 마일스톤 추가 모달은 현재 mock 수준입니다.
- 실제 생성 API가 들어오면 같은 모달 구조를 유지한 채 저장만 연결합니다.

### 업무 상세 커맨드
- 업무 시작, 담당자 지정, 검토 상신 같은 동작은 현재 mock 흐름을 포함합니다.
- `task` 도메인 API가 확장되면 현재 모달 구조를 유지한 채 매핑층만 교체합니다.

## 운영 원칙
- 메인 화면에는 실제로 연결 가능한 기능만 노출합니다.
- 미연동 기능은 코드에서 숨기고 이 문서에만 남깁니다.
- 프론트 수정 시 항상 백엔드 `dev`를 먼저 확인한 뒤 UI를 맞춥니다.
